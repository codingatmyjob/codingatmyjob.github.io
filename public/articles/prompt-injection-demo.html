Prompt Injection Challenge

This demo simulates a protected enterprise assistant with hidden instructions. Your objective is to use prompt-injection techniques to make the model reveal a protected value it was explicitly told not to disclose.

Everything runs fully in-browser with WebGPU. There is no backend, no API key, and no server-side filtering, which makes this a clean environment for testing prompt behavior directly.

How This Maps to OWASP Guidance

The exercise is mainly focused on OWASP LLM01 (Prompt Injection): untrusted user input attempts to override or bypass higher-priority instructions. It is intentionally simple so you can observe where instruction hierarchy breaks down.

Interactive Demo

Load the model, try a few attack patterns, and iterate quickly with reset. Temperature is adjustable to explore how sampling randomness can influence consistency in the model's refusals.

Load Environment

Start the prompt injection lab by loading the model into your browser.

Default model is tuned for fast loading. To switch models, edit the MODEL_ID constant in the page script.
